7 - HTML5 e CSS3

2 - Elementos semânticos

Elementos semânticos são aqueles que irão descrever claramente sua função tanto para o navegador quanto para o desenvolvedor.

Podemos ver elementos não-semânticos em <div> e <span>, pois eles não dizem nada a respeito do seu conteúdo, pode haver qualquer coisa lá dentro.

Por outro lado, podemos ver elementos emânticos em <form> e <table>, pois eles já especificam o que há dentro deles (dentro de um elemento table há uma tabela, e de um elemento form há um formulário).

Porém, no HTML4 não haviam elementos específicos para cada parte da página, tudo era dividido com divs, e então, a partir do HTML5 foram introduzidas tags justamente com a função de separar melhor o código, para que tanto o navegador quanto o desenvolvedor saibam exatamente sobre o que aquela divisão se trata. São os novos elementos semânticos do HTML5:

1) header: é utilizado para representar o cabeçalho de um documento ou seção declarado no HTML. Nele podemos inserir elementos de <h1> a <h6> até elementos para representar imagens, parágrafos ou mesmo listas de navegação.

2) section: é utilizado para representar uma seção dentro de um documento e geralmente contém um título, o qual é definido por meio de um dos elementos entre <h1> e <h6>. Podemos utilizar o section, por exemplo, para descrever as seções/tópicos de um documento. E ainda, segundo o W3C, "uma seção é um agrupamento temático de conteúdo, normalmente com um título."

3) article: é utilizado para declarar um conteúdo que não precisa de outro para fazer sentido em um documento HTML, por exemplo, um artigo em um blog. É recomendado identificar cada <article> com um título.

4) nav: é utilizado quando precisamos representar um agrupamento de links de navegação, que, por sua vez, são criados com os elementos <ul>, <li> e <a>.

5) aside: é utilizado para criar um conteúdo de apoio/adicional ao conteúdo principal, como por exemplo sidebars. O conteúdo do aside deve estar indiretamente relacionado ao conteúdo que o cerca.

6) main: é utilizado para especificar o conteúdo principal e, consequentemente, de maior relevância dentro da página. Para ser considerada bem construída, uma página deve apresentar apenas um conteúdo principal.

7) figure:é uma marcação de uso específico para a inserção de uma figura.

8) figcaption: é utilizado para incluir a descrição de uma figura. Deve estar sempre dentro do figure.

9) details: é utilizado para especificar detalhes adicionais que o usuário pode abrir ou fechar caso ele precise. O details comumente é criado como um widget interativo que o usuário pode abrir e fechar, sendo que por padrão ele vem fechado.

10) summary: é utilizado em conjunto com o details, e serve para definir um título para o details, fazendo com que esse título possa ser clicado para mostrar ou esconder conteúdo. Detalhe que o summary deve ser o primeiro elemento dentro do details.

11) footer: é utilizado para representar um rodapé de um documento, como a área presente no final de uma página web. Normalmente é utilizado para descrever informações de autoria, como nome e contato do autor, e data de criação do conteúdo.
