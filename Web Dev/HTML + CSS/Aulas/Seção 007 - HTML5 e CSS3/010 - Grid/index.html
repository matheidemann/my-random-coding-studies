10 - Display Grid

O display grid é um módulo complexo (e não uma única propriedade) focada na otimização da criação de grandes layouts responsivos para desktops e mobile.

Ele funciona junto do display-flex, atuando de forma macro, enquanto o flex atua de forma micro. Entretando, o ideal é utilizar ambos para atingir o melhor resultado possível de um design responsivo prático.

O display grid é um módulo complexo (e não uma única propriedade) focada na otimização da criação de grandes layouts responsivos para desktops e mobile.

Diferente do flex, o grid é bidimensional (possui duas dimensões), podendo trabalhar com linhas e colunas simultaneamente, enquanto o flex trabalha ou só com colunas, ou só com linhas.

Propriedades de contêiner e Propriedades de itens

O grid é divido em dois grupos, sendo que cada um dos grupos possui propriedades específicas:

- Grid container: é o contêiner com o display grid, sendo que dentro dele haverão outros elementos. Aqui são encontradas as propriedades:

Propriedades padrão

  • display
  • grid-template-columns
  • grid-template-rows
  • gap
    • row-gap
    • column-gap
  • grid-template-areas

Propriedades de alinhamento

  • justify-content
  • align-content

- Grid item: são os itens que estarão dentro de um grid containêr. Aqui são encontradas as seguintes propriedades:

Propriedades padrão

  • grid-column
    • grid-column-start
    • grid-column-end
  • grid-row
    • grid-row-start
    • grid-row-end
  • grid-area

Propriedades de alinhamento

  • justify-items
  • align-items
  • justify-self
  • align-self

Grid area

O grid area é um conjunto de propriedades e funções utilizadas para facilitar ainda mais a utilização do grid, adiconando a opção de criar uma estrutura completa do layout da página com nomes personalizados para cada célula do grid.

O grid area trabalha com as seguintes propriedades:

1) grid-template-areas: define o nome de cada célula do grid. Por exemplo, o documento pode possuir 3 colunas e 3 linhas, sendo que o header ocupará toda a primeira linha (grid-row: 1 /4) e apenas a primeira linha (grid-column: 1 / 2), e uma sidebar ocupará apenas uma coluna (grid-column: 1 / 2) e do fim do header ao fim da página (grid-row: 2 / 4).

Para visualizar melhor a propriedade, observe o esquema abaixo:

grid-template-areas: "header header header" "nav main main" "nav main main";;


2) grid-areas: define para cada célula do grid sua posição dentro da área criada no template. Ou seja, caso eu possua uma div que será o header, basta aplicar o valor "header" (caso assim esteja no grid-template-areas) e esta div automaticamente ficará da como foi definido no grid-template-areas.

Para visualizar melhor a propriedade, observe o esquema abaixo:

div1 = grid-area: header;

div2 = grid-area: nav;
