6 - PROPRIEDADES MAIS COMPLEXAS DE CSS

4 - Float

1) float: define como um elemento deve flutuar (literalmente). É como se a page tivesse duas folhas, e quando o elemento começa a flutuar, ele vai para a página de cima. O float compreende os valores "left", "right", "none" (valor padrão de todos os elementos).

Foi inicialmente criado para fazer com que elementos ficassem em volta (wraping) do elemento flutuando na esquerda ou na direita (como um texto em volta de uma imagem, sendo que, o texto ficaria ao lado da imagem, e após atingido o valor máximo de altura da imagem, o texto começaria ser escrito abaixado da imagem também).

Porém o float não é apenas usado para fazer com texto se envola em volta de uma imagem, e sim ele funciona para diversos layouts. Nos divs, por exemplo, que são elemento com o "display: block" (possuem por padrão uma largura de 100% da viewport), uma vez aplicado o float, esta div passará a ser um elemento com "display: inline-block", fazendo com que não ocupe mais 100% da viewport, e ocupando apenas o tamanho necessário para mostrar seu conteúdo.

Porém, o que seria feito caso eu não quisesse que o quadrado verde fosse para cima e ficasse atrás do quadrado vermelho? Bem, para solucionar isso, há uma propriedade que anda lado a lado com a propriedade float, e esta propriedade se da o nome de "clear".


2) clear: é utilizada para definir se elementos flutuantes poderão flutuar do lado de determinado elemento. Quando se utiliza o "clear" em um elemento, é como se fosse dito "não é permitido que nenhum elemento flutue do lado esquerdo ou direito deste elemento". O clear compreende os valores "left" (nenhum elemento flutuante poderá ficar à esquerda do elemento), "right" (nenhum elemento flutuante poderá ficar à direita do elemento) e "both" (nenhum elemento flutuante poderá ficar à esquerda ou direita do elemento).

Exemplo (apenas o quadrado vermelho está com "float: left;"). É exatamente igual o último exemplo demonstrado (sem a margem de 30px), porém, desta vez, o quadrado verde está com a propriedade "clear: left", que significa que nenhum elemento flutuante poderá ficar ao lado dele:



Ainda, vale destacar que um elemento com a propriedade "clear: left" ou "clear: right" ainda pode ser um elemento flutuante, e fazer com que algum elemento fique abaixo dele.

Exemplo (o quadrado vermelho está com "float: left;", o quadrado verde está com "clear: left" e "float: left;"). Neste exemplo é possível visualizar o que foi dito logo acima, pois o quadrado azul desapareceu, uma vez que ficou em baixo de um elemento flutuante (o quadrado verde), e o quadrado verde não ficou em baixo do quadrado vermelho, pois sua propriedade de clear não permite elementos flutuantes a sua esquerda (foi adicionada uma margin-left de 30px no quadrado azul para poder ser visualizado):









Outro detalhe importante é sobre o comportamento de uma div com texto dentro. A div que está por baixo de um elemento flutuante ficará literalmente atrás do elemento flutuante, portanto, a cor de fundo também ficará por trás, porém o texto não, o texto ainda assim ficará em volta do elemento flutuante, mesmo que seu fundo esteja sendo parcialmente coberto pelo mesmo.

Esta correção ocorreu pelos simples fato de que: antes a imagem estava sendo aplicada apenas para o elemento de parágrafo, pois a div que engloba todas as outras divs estava trabalhando apenas com elementos que estavam na "folha de baixo", portanto, ignorando a existência da altura dos quadrados. Porém, quando foi adicionada a propriedade "overflow: auto;" na div principal, logo foi considerada a altura dos quadrados, pois agora a "folha de cima" está sendo levada em conta também, e como seu overflow é de valor "auto", sua altura será automática, isto é, irá se basear na altura máxima presente dentro da div principal, que no caso era de 100px dos quadrados.
